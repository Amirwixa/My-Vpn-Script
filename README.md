# Backhaul Manager (bhmgr) — نسخه بازنویسی‌شده

بازنویسی کامل اسکریپت [Azumi67/Backhaul_script](https://github.com/Azumi67/Backhaul_script)
با هدف رفع باگ‌ها و ساده‌تر کردن کار با آن.

## چرا بازنویسی شد؟

اسکریپت اصلی (`backhaul.py`) حدود **۴۷ هزار خط و ۸۲۳ تابع** داشت که بخش زیادی از آن
کد کاملاً تکراری بود (مثلاً ۱۰ تابع جدا برای uninstall شماره ۱ تا ۱۰، برای هر پروتکل
و هر نقش جداگانه). این باعث چند مشکل واقعی می‌شد:

| مشکل در نسخه‌ی اصلی | راه‌حل در این نسخه |
|---|---|
| منوی اصلی `0. Exit` نشان می‌داد ولی فقط `"0."` (با نقطه) را قبول می‌کرد → گزینه‌ی خروج عملاً کار نمی‌کرد | همه‌ی گزینه‌ها یکدست و بدون نقطه |
| حالت Single همیشه در مسیر ثابت `server.toml` / `client.toml` ذخیره می‌شد → ساخت تانل دوم، تانل اول را **پاک می‌کرد** | هر تانل یک نام یکتا و فایل/سرویس مخصوص خودش دارد؛ تعداد تانل نامحدود |
| نسخه‌ی باینری Backhaul هاردکد بود (v0.6.5) و هیچ‌وقت آپدیت نمی‌شد | نسخه به‌صورت خودکار از GitHub API گرفته می‌شود و امکان آپدیت دستی هم هست |
| اکثر ورودی‌ها validation نداشتند → با یک عدد اشتباه کل برنامه کرش می‌کرد | همه‌ی ورودی‌ها بررسی می‌شوند و در صورت اشتباه دوباره می‌پرسد |
| حذف چندگانه از طریق ۲۰ تابع کپی‌شده (`uninstall_multi_iran1`...`10`, `kharej1`...`10`) | یک فرم واحد که می‌توانید چند شماره را با کاما انتخاب کنید یا `all` بزنید، با تأیید قبل از حذف |
| تایمر ری‌استارت با یک اسکریپت bash دستی (`while true; sleep`) پیاده شده بود | با `systemd timer` استاندارد که با `systemctl status` قابل مشاهده است |

## نصب

```bash
sudo bash install.sh
```

یا مستقیم (بعد از آپلود `bhmgr.py` روی گیت‌هاب خودتان):

```bash
sudo bash -c "$(curl -fsSL https://raw.githubusercontent.com/Amirwixa/My-Vpn-Script/refs/heads/main/install.sh)"
```

## استفاده

```bash
sudo bhmgr
```

از منوی اصلی می‌توانید:
1. **ساخت تانل جدید** — سرور ایران یا کلاینت خارج، با هر ۷ نوع transport (TCP, TCP Mux, WS, WSS, WS Mux, WSS Mux, UDP)
2. **وضعیت تانل‌ها** — لیست همه‌ی تانل‌ها با وضعیت زنده (active/inactive) و مشاهده‌ی لاگ زنده
3. **ویرایش تانل** — تغییر تنظیمات یک تانل موجود بدون نیاز به ساخت از صفر
4. **حذف تانل** — حذف یک یا چند تانل با تأیید قبل از حذف
5. **مانیتورینگ ترافیک** — دیدن حجم و تعداد بسته‌های عبوری از یک تانل با tcpdump
6. **آپدیت باینری Backhaul** — بررسی و دانلود آخرین نسخه‌ی Backhaul

## ساختار فایل‌ها روی سرور

```
/etc/backhaul-manager/
  ├── tunnels.json          ← رجیستری تمام تانل‌ها
  ├── configs/<name>.toml   ← کانفیگ هر تانل
  └── certs/                ← گواهی‌های self-signed برای WSS
/usr/local/bin/backhaul/backhaul   ← باینری Backhaul
/etc/systemd/system/backhaul-<name>.service   ← سرویس هر تانل
/etc/systemd/system/backhaul-<name>-reset.timer  ← تایمر ری‌استارت (اختیاری)
```

## نکته درباره‌ی آدرس دانلود در install.sh

مقدار `SCRIPT_URL` در `install.sh` باید به ریپوی گیت‌هاب خودتان اشاره کند (بعد از
اینکه `bhmgr.py` را در ریپوی خودتان آپلود کردید). اگر فقط می‌خواهید محلی تست کنید،
کافیست هر دو فایل `bhmgr.py` و `install.sh` را کنار هم روی سرور بگذارید و
`sudo bash install.sh` را اجرا کنید — در این حالت از فایل محلی استفاده می‌کند.
